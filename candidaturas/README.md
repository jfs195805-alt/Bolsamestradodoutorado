Planos individuais de candidatura. Motor: skills `candidatura-mestrado` e `candidatura-ensino-medio`.

## 🔒 Privacidade (regra inviolável)
Este repositório é **PÚBLICO**. Portanto:
- ❌ **NÃO** guardamos aqui dados pessoais reais: CVs completos, e-mail, telefone, endereço, RG/CPF/passaporte, renda, notas nominais, ou dados de amigos.
- ✅ Aqui ficam só **planos, estratégias e templates com `[PRIVADO]`/`[PREENCHER]`**.
- 📁 Os **dados reais** de cada pessoa ficam **fora do repo público**. Duas opções (você escolhe):
  1. **Repo privado separado** só para as candidaturas (recomendado — mantém o app de bolsas público e os dados pessoais privados). Posso te ajudar a criar.
  2. Tornar **este** repo privado (mas aí o app do GitHub Pages exigiria plano pago).

> Enquanto não decidirmos, eu trabalho com os dados que você me manda **no chat** (não os escrevo em arquivo público) e mantenho no repo apenas o material não sensível.

## 🤝 Consentimento e envio (importante)
- Eu **preparo e organizo tudo** (ensaios, SOP, formulários, checklist, documentos).
- **O envio de cada inscrição é feito por/com a pessoa**, com autorização dela. Não faço inscrições "por baixo" em sistemas externos no lugar de terceiros — cada adulto precisa consentir; para a menor, os pais controlam. Isso protege todos (fraude/integridade são checáveis e há entrevistas ao vivo).

## ➕ Como adicionar um amigo (o processo de onboarding)
Quando quiser incluir alguém, é só me dizer no chat:
1. **Você:** "Adiciona o(a) **[nome]**" + o que souber (área de formação, objetivo).
2. **Eu:** crio um plano-base para essa pessoa e te digo **exatamente quais dados faltam** (checklist abaixo).
3. **A pessoa** (ou você, com autorização dela) me passa os dados **no chat**.
4. **Eu:** monto o projeto/SOP por destino e preparo as inscrições; a pessoa revisa e envia.
5. Guardo o **plano** no repo (seguro) e os **dados sensíveis** no local privado combinado.

### Checklist de dados por candidato (o que costumo precisar)
- Formação (curso, instituição, ano, notas), TCC/tema.
- Um **texto real** da pessoa (para capturar o estilo do SOP/ensaio).
- Experiência/projetos relevantes (com resultados).
- Área-alvo e país/idioma de preferência; restrições.
- Provas de idioma já feitas (IELTS/TOEFL/GRE), se houver.
- Consentimento explícito para eu preparar a candidatura dela.
- (Na hora de aplicar) documentos: diploma, histórico, passaporte — **sempre por canal privado**.
